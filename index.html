Automated Background Report Delivery

  • Problem: Manual report generation and distribution were time-consuming, prone to errors, and lacked consistency. Stakeholders often received delayed reports, affecting timely decision-making.
  • Solution: Developed a classical ABAP report that runs in background, creates a spool request from WRITE statements, converts the spool output to PDF using CONVERT_ABAPSPOOLJOB_2_PDF, and automatically sends the PDF via email using SO_DOCUMENT_SEND_API1. Added features to delete the spool post-delivery and trigger immediate dispatch with RSConn01. Scheduled the report as a background job (SM36) for fully automated execution.
  • Impact: Achieved 100% automation of report generation and delivery, ensuring timely and accurate PDF distribution, reducing manual effort, and improving stakeholder satisfaction.

Tech Stack: ABAP, Classical Reports, Spool Management, PDF Conversion, Background Jobs
